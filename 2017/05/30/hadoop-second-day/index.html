Hadoop的dfs分布式文件系统的介绍

HDFS的shell和linux的shell操作基本一样

HDFS的shell操作

  1. 查看帮助
    hadoop fs -help <cmd>
  2. 上传
    hadoop fs -put <linux上文件> <hdfs上的路径>
  3. 查看文件内容
    hadoop fs -cat <hdfs上的路径>
  4. 查看文件列表
    hadoop fs -ls /
  5. 下载文件
    hadoop fs -get <hdfs上的路径> <linux上文件>

HDFS的架构

HDFS架构由三个角色组成，分别是NameNode,DataNode,Secondary NameNode;各自的作用分别是

  1. NameNode。管理者，管理DataNode干活
  2. DataNode。存贮数据
  3. Secondary NameNode。可以理解为NameNode的助理，帮助NameNode干活。它并不是NameNode的热备，不能代替NameNode干活，NameNode宕机，也不会自动切换到它上面工作，并且NameNode宕机后它也不工作了。
